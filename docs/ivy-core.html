Typography

Headings use fluid sizing via clamp() — they scale smoothly between a minimum and maximum size. Body text uses --lh: 1.6 for comfortable reading rhythm, with a max content width of --measure: 64rem.

H1 — Page title clamp(1.875rem, 2vw, 2.5rem)

H2 — Section heading clamp(1.5rem, 1.5vw, 2rem)

H3 — Subsection clamp(1.25rem, 1vw, 1.5rem)

H4 — Minor heading (1.125rem)

H5 — Small heading (1rem)
H6 — Micro heading (0.875rem, muted color)

A body paragraph. Ivy sets a comfortable line height of 1.6, a readable font stack (Inter or system-ui), and consistent spacing between block elements. Inline bold, italic, code, and links all have clean, accessible defaults. Small text inherits muted color and 87.5% size.

A horizontal rule (<hr>) uses a 1px border rendered with --color-border, with --space-xl (2rem) vertical margin above and below.

Links & Navigation

Links are underlined with a 2px offset and use --color-link. On focus, they show a solid outline in the same color. A link to an external page works the same way.

nav — flex link list

A <nav> wrapping a <ul> or <ol> renders as a horizontal flex row. Links inside nav have no underline by default; underline appears on hover.

  • Home
  • Current page
  • About
  • Contact

The aria-current="page" attribute makes the active item bold. This is handled purely via CSS.

header nav

Links inside a header nav receive a slightly larger font size (1rem) for improved tap target size.

Code & Preformatted

Inline code

Use <code> for inline code snippets like const x = 1 or CSS property names like --color-primary. It gets a surface background, border, and monospace font.

Code block

Wrap <code> inside <pre> for multi-line blocks. The inner <code> has its background and border removed when nested inside <pre>.

// JavaScript example
function greet(name) {
  const msg = `Hello, ${name}!`;
  console.log(msg);
  return msg;
}

Keyboard key

Use <kbd> for keyboard shortcuts. Press ⌘ + K to open the command palette. Or use Ctrl + Shift + P to open the command palette in VS Code. When using ivy.extra.css, <kbd> gets an extra bottom border for an embossed key appearance.

Samp

The <samp> element, used for sample program output: Error: file not found at path /foo/bar. It shares the monospace font stack.

Media

Images, video, canvas, and SVG all get max-width: 100%; height: auto to prevent overflow. Images are display: block; SVGs are inline-block.

Figure with caption

Video

Video also respects max-width: 100%.

iframe

Iframes get max-width: 100% and border: 0.

Tables

Tables are full-width with border-collapse: collapse. Cells have --space-sm padding and a bottom border. <thead th> gets a thicker bottom border to separate header from body.

Forms

All form controls inherit the body font and line height. Inputs and selects are full-width with consistent padding, border, and border-radius from the token set. Buttons use the primary color and switch to a lighter tint on hover.

Validation states

Use aria-invalid="true" on inputs that fail validation. The border changes to a --danger-tinted color. Use .form-error for the error message and .form-help for neutral hint text.

Accessibility

Text selection

Select some text on this page — the selection background uses a 25% tint of --color-link so it looks on-theme in both light and dark modes. Try it: select this sentence to see the selection color.

Skip link

The "Skip to content" link at the top of the page is visually hidden with .sr-only until it receives focus. Tab to the top of the page to see it appear. It's implemented with both .sr-only and .sr-only-focusable:

<a class="sr-only sr-only-focusable" href="#content">Skip to content</a>

Screen reader only text

The .sr-only class hides content visually while keeping it available to screen readers. Useful for icon buttons, form hints, and decorative separators.

<button>
  <svg aria-hidden="true">...</svg>
  <span class="sr-only">Close dialog</span>
</button>

The text below is present in the DOM but visually invisible:

This text is only visible to screen readers.

(Nothing visible above — that's correct.)

Forced colors (Windows High Contrast)

Inside @media (forced-colors: active), Ivy restores OS system colors on all controls and removes custom borders that would otherwise disappear. The custom select caret is replaced with the native OS caret.

Reduced motion

When prefers-reduced-motion: reduce is active, all transitions and animations are set to 0.01ms duration and a single iteration. No motion plays at all.

Print

When printing, Ivy hides navigation, forces a white background and black text, ensures all links are underlined, and wraps pre/code content.
