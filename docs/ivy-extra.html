Details / Summary

Native <details> disclosure elements get a border, border-radius, and surface-2 background. Stacked <details> elements get automatic top margin. An open panel switches to surface-3.

What is Ivy Extra?

An optional polish layer over the classless core. It adds component-level styling through data-* attributes and semantic class names — no extra markup needed.

When should I use Extra vs Core?

Use Core alone for blogs, documentation sites, and content pages. Add Extra when you need interactive components: modal dialogs, tooltips, styled cards, or form controls like switches and range inputs.

Open by default

Add the open attribute on <details> to show it expanded on page load.

View HTML

  <p>Content shown when expanded.</p>
</details>

  <summary>Already open</summary>
  <p>Visible on load.</p>
</details>

Tooltips

CSS-only tooltips. Add data-tooltip="…" to any element. The tooltip appears on hover and :focus-within. Use data-placement to control direction.

View HTML 
<!-- Default (top) -->
<button data-tooltip="Tooltip text">Top</button>

<!-- Right -->
<button data-tooltip="Tooltip text" data-placement="right">Right</button>

<!-- Bottom -->
<button data-tooltip="Tooltip text" data-placement="bottom">Bottom</button>

<!-- Left -->
<button data-tooltip="Tooltip text" data-placement="left">Left</button>
Accessibility note

The data-tooltip content is rendered via CSS content: attr(data-tooltip) and is not reliably accessible to screen readers. For interactive elements, supplement with aria-label or link to a visible/sr-only element via aria-describedby:

<button
  data-tooltip="Removes the item permanently"
  aria-label="Delete item — removes permanently">
  Delete
</button>

Dialog

Native <dialog> modal with Ivy Extra styles: border, padding, shadow, and a frosted-glass backdrop. The dialog content should be wrapped in an <article>.

Modal title

This is a native <dialog> styled by Ivy Extra. It centers automatically and adapts to viewport width on small screens.

The backdrop uses color-mix() for a dark tint and backdrop-filter: blur() for a frosted-glass effect.
View HTML 
<!-- Trigger -->
<button id="openDialog">Open dialog</button>

<!-- Dialog (hidden until showModal() is called) -->
<dialog id="myDialog">
  <article>
    <h3>Title</h3>
    <p>Content here.</p>
    <div class="btn-group" data-align="end">
      <button id="closeDialog" data-variant="ghost">Cancel</button>
      <button data-variant="primary">Confirm</button>
    </div>
  </article>
</dialog>
Requires JavaScript

The native <dialog> element must be opened with dialog.showModal() (for a proper modal with backdrop and focus trap) or dialog.show() (for a non-modal). Close it with dialog.close().

const dialog = document.getElementById('myDialog');

// Open as a modal (recommended — traps focus, adds backdrop)
document.getElementById('openDialog')
  .addEventListener('click', () => dialog.showModal());

// Close
document.getElementById('closeDialog')
  .addEventListener('click', () => dialog.close());

// Optional: close on backdrop click
dialog.addEventListener('click', e => {
  if (e.target === dialog) dialog.close();
});
Accessibility

Using showModal() automatically traps focus inside the dialog and restores it to the trigger element when closed. Add aria-labelledby pointing to the dialog's heading for screen reader context:

<dialog aria-labelledby="dialog-title">
  <article>
    <h3 id="dialog-title">Title</h3>
    ...
  </article>
</dialog>

Accessibility note

Always include fallback text inside <progress> for browsers that don't support the element. Add aria-valuenow, aria-valuemin, aria-valuemax, and aria-label for screen reader context:

<progress
  max="100"
  value="32"
  aria-label="Upload progress"
  aria-valuenow="32"
  aria-valuemin="0"
  aria-valuemax="100">32%</progress>

Form Enhancements

Extra adds accent colors, a toggle switch, a fully styled range input, and a matched file upload button.

Checkbox and radio — accent color

Checkboxes, radio buttons, and native progress bars use accent-color: var(--color-primary). Add data-variant="secondary" for the secondary accent color.

  

  

Toggle switch

Use <input type="checkbox" role="switch"> for a toggle switch. The switch animates on check/uncheck.

View HTML 
<label>
  <input type="checkbox" role="switch" checked>
  Enable notifications
</label>
Accessibility

role="switch" communicates the on/off semantics to screen readers. The checked state is announced as "on" and unchecked as "off". Always pair with a visible label.

Range input

Extra styles the track and thumb. The filled portion of the track requires a small JS snippet to set the --range-progress CSS custom property on the element.

View HTML + required JavaScript 
<input id="myRange" type="range" min="0" max="100" value="60">

<script>
  function initRange(el) {
    function update() {
      const min = parseFloat(el.min) || 0;
      const max = parseFloat(el.max) || 100;
      // Set as a dimensionless ratio (0–1), not a percentage
      const ratio = (parseFloat(el.value) - min) / (max - min);
      el.style.setProperty('--range-progress', ratio);
    }
    update();                              // set on init
    el.addEventListener('input', update); // update on drag
  }

  // Initialize all ranges on the page
  document.querySelectorAll('input[type=range]').forEach(initRange);
</script>
How it works

The styled track is a CSS linear-gradient background that uses --range-progress as the width of the filled segment. The filled color is a 40% tint of --color-primary. Firefox natively supports ::-moz-range-progress so the JS is mainly needed for WebKit/Blink.

Breadcrumbs

A <nav aria-label="breadcrumb"> wrapping an <ol>. Separators are added automatically via CSS (no extra markup). The current page is marked with aria-current="page".

  1. Home
  2. Library
  3. Components
  4. Breadcrumbs
View HTML 
<nav aria-label="breadcrumb">
  <ol>
    <li><a href="/">Home</a></li>
    <li><a href="/library">Library</a></li>
    <!-- Last item: current page, no link -->
    <li aria-current="page">Current page</li>
  </ol>
</nav>
Accessibility

The aria-label="breadcrumb" on the <nav> identifies it to screen readers. aria-current="page" on the last item tells screen readers it represents the current location. The CSS-generated > separator is purely decorative (pointer-events: none) and not read aloud.

Badges

Notification chip overlaid on an element. Requires both aria-label (for screen reader context) and data-badge (for the chip value) on the same element.

View HTML 
<!-- Primary badge (default) -->
<button aria-label="Inbox, 4 new messages" data-badge="4">
  Inbox
</button>

<!-- Secondary badge color -->
<button
  aria-label="Messages, 99+ unread"
  data-badge="99+"
  data-tone="secondary">
  Messages
</button>
Both attributes required

The badge selector is [aria-label][data-badge] — it only activates when both attributes are present. This enforces accessible labeling: the aria-label should include the badge content in a meaningful way (e.g. "Inbox, 4 new messages" not just "Inbox").

Busy / Spinner

Adding aria-busy="true" to any element shows a small CSS spinner in its bottom-right corner using a ::after pseudo-element. The element must have position: relative (automatically applied by Ivy Extra).

On a button

On a card

Content loading…

On any container

Processing your request…
View HTML 
<!-- On a button -->
<button aria-busy="true">Saving…</button>

<!-- On a card -->
<div class="card" aria-busy="true" style="min-height: 4rem;">
  Content loading…
</div>
Implementation notes

The spinner renders on ::after, positioned absolutely in the bottom-right corner. The container needs sufficient height to show the spinner. Customize size and thickness with CSS variables: --spinner-size (default 1.1rem) and --spinner-thickness (default 3px).

Toggle aria-busy via JavaScript when your async operation starts and ends. This simultaneously shows/hides the spinner and communicates loading state to screen readers.

el.setAttribute('aria-busy', 'true');
await doAsyncWork();
el.removeAttribute('aria-busy');

Cards & Boxes

Two container variants: .card has a shadow and a hover lift; .box is flat with just a border. Both share the same padding, border-radius, and surface background.

Box vs Card

Box

Flat container — border and border-radius only. No shadow, no hover effect. Use for static grouping.

Card

Adds a subtle shadow. Hover over this card to see the shadow deepen and the border shift to the primary color.

Card with header and footer

Add a <header> and/or <footer> as a direct child of .card. They bleed to the card edges with negative margins and get a border to separate them from the body.

Card title

Card body content. This area has standard padding and can hold any content — text, images, forms, etc.

View HTML — card with header/footer 
<div class="card">
  <header>
    <h4>Card title</h4>
  </header>
  <p>Body content here.</p>
  <footer>
    <div class="btn-group" data-align="end">
      <button data-variant="ghost" data-size="sm">Cancel</button>
      <button data-variant="primary" data-size="sm">Save</button>
    </div>
  </footer>
</div>

Callouts

Contextual message blocks. Apply a tone with data-tone="…" or a matching class name. A <strong> as the first child renders as a block label.

Neutral

Default callout — no tone attribute. Uses --surface-2 background.

Info

Use data-tone="info" or class="callout info" for informational messages.

Success

Use data-tone="success" or class="callout success" for positive outcomes.

Warning

Use data-tone="warn" or class="callout warn" for cautionary messages.

Danger

Use data-tone="danger" or class="callout danger" for errors or destructive actions.

View HTML 
<!-- Neutral -->
<div class="callout">
  <strong>Note</strong>
  <p>Content here.</p>
</div>

<!-- Toned — use data-tone attribute OR matching class -->
<div class="callout" data-tone="info">...</div>
<div class="callout" data-tone="success">...</div>
<div class="callout" data-tone="warn">...</div>
<div class="callout" data-tone="danger">...</div>

<!-- Class syntax (equivalent) -->
<div class="callout info">...</div>
<div class="callout success">...</div>
