Ivy CSS

A tiny, classless CSS baseline that makes semantic HTML look great out of the box.
Add the optional extras layer for buttons, cards, tooltips, and more — styled with data-* attributes, not class soup.

Get started Core demo Extra demo GitHub ↗

Two layers

Ivy is split into two files so you only ship what you need.

Core — ivy.css

Classless defaults for every HTML element. Zero classes required — just write clean semantic HTML.

  • Fluid headings with clamp()
  • Automatic light/dark mode
  • Forms, tables, code, media
  • Focus-visible, sr-only, skip link
  • Forced-colors and print support
View core demo →

Extra — ivy.extra.css

Optional polish and components, activated with data-* attributes or small semantic class names.

  • Button variants and sizes
  • Cards, boxes, callouts
  • Tooltips, dialog, progress
  • Switches, range inputs, badges
  • Shadows, surfaces, spinner
View extra demo →

Installation

Option 1 — Core only

Classless defaults, nothing else. Great for blogs, docs, and content-focused sites.

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/rlnorthcutt/ivy/dist/ivy.min.css">

Option 2 — Core + Extra

Add the optional polish layer for buttons, cards, and components.

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/rlnorthcutt/ivy/dist/ivy.min.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/rlnorthcutt/ivy/dist/ivy.extra.min.css">

Option 3 — Full bundle

Core + Extra in one file. Simplest option for most projects.

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/rlnorthcutt/ivy/dist/ivy.full.min.css">

Option 4 — With Lattice layout

Add Lattice for a minimal layout utility layer (grids, flex, spacing, positioning).

<link rel="stylesheet" href="ivy.full.min.css">
<link rel="stylesheet" href="lattice.full.min.css">

Theme configuration

Ivy ships with both a light and a dark palette. You choose how to activate them.

1. System preference (default)

Ivy automatically follows prefers-color-scheme when no manual override is set. No HTML required — it just works.

2. Data attribute (recommended for JS control)

Set data-theme on the <html> element to force a theme. The attribute overrides the system preference.

<html data-theme="dark">   <!-- force dark -->
<html data-theme="light">  <!-- force light -->
<!-- remove the attribute to return to system preference -->

3. CSS class

Alternatively, toggle the .dark or .light class on <html>.

<html class="dark">   <!-- force dark -->
<html class="light">  <!-- force light -->

4. Dark mode toggle web component

The bundled <dark-mode-toggle> web component renders a theme switch button that stores the user's preference in localStorage.

<!-- Place anywhere in your layout -->
<dark-mode-toggle></dark-mode-toggle>

<!-- Load the script -->
<script type="module" src="dark-mode-toggle.min.js" defer></script>
Toggle component options

The toggle defaults to strategy="attr", which sets data-theme on <html>. Use strategy="class" to toggle a class instead. Customize the button appearance with CSS variables: --track-width, --track-height, --track-bg-light, --track-bg-dark, --thumb-bg.

Disabling responsive type scaling

Override --font-size to a fixed value in your own stylesheet. Because Ivy rules live in @layer ivy-base, your unlayered CSS wins without !important:

/* Opt out of responsive type scaling */
:root { --font-size: 100%; }

Customization

Ivy is built entirely on CSS custom properties. Override any token in your own stylesheet. Because all Ivy rules are inside @layer ivy-base and @layer ivy-extra, your unlayered rules always take precedence — no !important required.

How cascade layers work

<!-- Ivy styles live in @layer ivy-base and @layer ivy-extra -->
<link rel="stylesheet" href="ivy.full.min.css">

<!-- Your unlayered CSS beats any layered rule automatically -->
<style>
  /* This wins even though ivy.css sets it */
  h1 { font-weight: 400; letter-spacing: -.02em; }
  button { border-radius: 999px; }
</style>

Variable override pattern

Override the palette tokens (the ones ending in -light and -dark) to customize both themes at once.

Override palette tokens, not applied tokens

Override --color-primary-light and --color-primary-dark, not --color-primary directly. The applied token gets remapped by the theme-switching logic, so overriding it alone breaks dark mode.

<style>
  :root {
    /* Primary brand color */
    --color-primary-light: #7c3aed;
    --color-primary-dark:  #a78bfa;
    --color-primary-text-light: #ffffff;
    --color-primary-text-dark:  #1a0030;

    /* Link color (often same as primary) */
    --color-link-light: #7c3aed;
    --color-link-dark:  #a78bfa;

    /* Secondary action color */
    --color-secondary-light: #0369a1;
    --color-secondary-dark:  #38bdf8;
    --color-secondary-text-light: #ffffff;
    --color-secondary-text-dark:  #0c1a2e;

    /* Typography */
    --font-sans: 'Your Font', system-ui, sans-serif;
    --lh: 1.75;        /* line height */
    --measure: 72rem;  /* max content width */

    /* Spacing — scales all derived spacing tokens */
    --m: 1.125rem;

    /* Shape */
    --radius: 0.5rem;
  }
</style>

Browser support

Ivy uses modern CSS features. All are baseline-supported in every current browser (Chrome, Firefox, Safari, Edge).

FeatureUsed for
@layerCascade layers so your CSS wins without !important
:focus-visibleFocus rings only for keyboard navigation
prefers-color-schemeAutomatic dark mode
color-mix()Button hover states, callout tints, border blends
clamp()Fluid heading sizes
accent-colorCheckbox/radio/range primary color
<dialog>Native modal (ivy.extra.css)
backdrop-filterDialog backdrop blur (ivy.extra.css)
forced-colorsWindows High Contrast / OS accessibility modes

Accessibility

  • Focus visible: Ivy uses :focus-visible so focus rings appear only during keyboard navigation, not on mouse click.
  • Skip link: Add <a class="sr-only sr-only-focusable" href="#content">Skip to content</a> as the first element in <body>. It becomes visible on focus.
  • Screen reader only text: Use class="sr-only" to visually hide content that should still be read aloud.
  • Forced colors: Ivy includes a @media (forced-colors: active) block that restores OS system colors and border visibility in Windows High Contrast mode.
  • Reduced motion: All transitions and animations are disabled when prefers-reduced-motion: reduce is active.
  • Print: Navigation is hidden, backgrounds become white, and links are underlined for print output.
