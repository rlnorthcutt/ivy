CSS micro-stack

Ivy (classless) + Lattice (layout)

Write clean, semantic HTML with Ivy. Add tiny, semantic positioning with Lattice. Sprinkle a few custom rules and ship.

Get started GitHub

Why this stack?

Classless first

Ivy styles semantic HTML (headings, forms, tables, media) with a small, elegant baseline.

Tiny layout

Lattice provides minimal, readable utilities for grids, spans, and placement. No framework lock-in.

Composable

Use Ivy alone for blogs & docs. Add Lattice when you need responsive structure or two-column layouts.

Quick start

Add Ivy (classless) and Lattice (layout) to your page:

<!-- Classless theme -->
<link rel="stylesheet" href="/path/to/ivy.css">
<!-- Optional polish -->
<link rel="stylesheet" href="/path/to/ivy.extra.css">

<!-- Layout utilities -->
<link rel="stylesheet" href="/path/to/lattice.min.css">
<!-- Optional full coverage (core + extra) -->
<link rel="stylesheet" href="/path/to/lattice.full.min.css">

Tip: Use lattice.full.min.css if you want one file; otherwise include just what you need.

Minimal example

<main class="l-place-content">
  <article>
    <h1>Hello, Ivy</h1>
    <p>This is clean HTML with elegant defaults.</p>
    <section class="grid sm-col-2 gap-3">
      <div class="demo-card">Card A</div>
      <div class="demo-card">Card B</div>
    </section>
  </article>
</main>

Files & builds

The CI builds minified files into dist/. Use a single file or compose them.

FileWhat
lattice.min.cssLattice core (lean layout utilities)
lattice.extra.min.cssExtras for full coverage
lattice.full.min.cssCore + Extra in one file
ivy.cssClassless base (typography, forms, tables)
ivy.extra.cssOptional polish (buttons, dialog, tooltips…)

Note: If Pages serves dist/ as root, copy ivy*.css into dist/ or update paths.

Philosophy

  • Classless first: Author HTML, not frameworks.
  • Utilities, not systems: Lattice adds just enough positioning.
  • Small layers: Core → Extra → a pinch of custom CSS.
